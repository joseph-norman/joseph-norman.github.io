On starting a technical blog

I’ve been trying back and forth for the last few years in deciding on what kind of technical blog I want to write. That is - What do I want to write about?

The truth is I’m not sure. I’m leaning towards writing content while I learn cool shit, so for now I’ll stick with that. I’m using Hexo to generate my blog, and an awesome theme (I’d write my own but my design skills are laughable at best). For now this combination is working (while also serving the content on Github pages).

I’m using a mixture of Hexo & Github pages mainly for its simplicity. I’ve looked at a number of solutions out there (including my own, which is a project in itself) but ultimately it came down to a few key points which I’m sticking to:

  1. How easy is it to use? (How easy is it to go from zero -> customised programming blog?)
  2. Can I customise it? (I’m terrible at design, so is there a good blog theme I can easily use?)
  3. How is it to maintain? (i.e tracking changes, publishing posts, making amendments)

I’m a JS developer mainly at the time of writing this blog, which was my main reason in choosing Hexo. Staying in my own comfort zone.

Let’s begin!
